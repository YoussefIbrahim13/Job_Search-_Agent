## вңҰ What Is This?

**RecruitBot** is an autonomous recruitment AI agent that finds *real, currently open* job postings tailored to a candidate's skill stack вҖ” from a live job search or a raw CV upload. It doesn't just scrape; it reasons, filters, scores, and validates every listing through a 5-layer quality pipeline before surfacing results.

Built on **LangGraph**, powered by **Groq's LLaMA 3.3 70B**, and backed by **Tavily** for structured web search across 11 approved job boards, it handles everything from query construction to live staleness detection вҖ” including Arabic posting-age strings.

```
в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```

### The Agent State Machine

The LangGraph graph compiles a deterministic state machine with four nodes:

| Node | Role |
|---|---|
| `llm_node` | Drives Groq's LLaMA 3.3 70B with tool bindings; issues parallel Tavily calls |
| `tool_node` | Executes `tavily_job_search` + `scrape_job_page` tools |
| `coerce_internship_node` | Injects a follow-up message if the internship query was skipped |
| `graceful_exit_node` | Extracts + validates JSON; falls back to ToolMessage recovery (FIX R) |

## вңҰ Feature Highlights

### рҹ”Қ Dual-Mode Search
- **CV Upload** вҖ” Accepts PDF, DOCX, or TXT. Skills are extracted deterministically via a pre-compiled regex vocabulary of 100+ tech terms before being injected into the LLM prompt. No hallucinated skills.
- **Targeted Search** вҖ” Direct job title + location query. The agent constructs valid Tavily queries from inline tech signals in the title.

### рҹ§  Intelligent Query Construction
The system prompt enforces strict query grammar: `<TECH> <ROLE> <LOCATION> <modifier> site:<BOARD>`. A sanitiser strips natural-language prefixes ("please find me...", "search for...") before the query reaches Tavily, and a stray-token stripper prevents Go-language searches from being accidentally injected into `.NET` queries.

### рҹӣЎпёҸ 5-Layer Anti-Zombie Pipeline
Every result from Tavily passes through:
1. **Domain blacklist** вҖ” Known CSR zombie aggregators (GulfTalent, NaukriGulf, etc.)
2. **Content pollution filter** вҖ” Quora, Reddit, Medium, Stack Overflow, and 30+ more
3. **Positive-Assertion Path Gate** вҖ” Per-board canonical URL regex validation (e.g. Wuzzuf `/jobs/p/<slug>`, LinkedIn `/jobs/view/<id>`)
4. **Snippet staleness scan** вҖ” Detects `"Posted 5 years ago"`, `"Щ…ЩҶШ° 4 ШіЩҶЩҲШ§ШӘ"`, and closed/filled declarations in both English and Arabic
5. **Live shallow probe** вҖ” Fetches the first 8KB of the raw HTML for Wuzzuf and LinkedIn listings, checks for closure badges before sidebar noise corrupts the signal

### рҹ”’ Upload Security
- **File size guard** вҖ” Rejects uploads over 10 MB before the body is fully read
- **Magic-bytes validation** вҖ” Cross-checks file content against declared extension; rejects executables (ELF, PE, Mach-O, shebang scripts) regardless of filename
- **Binary text detection** вҖ” NUL-byte sniffing and binary ratio heuristic for `.txt`/`.md` uploads

### рҹ“Ҡ Match Scoring (0вҖ“100)
Each job listing receives a differentiated score:

| Component | Max Points | Criteria |
|---|---|---|
| `TITLE_MATCH` | 50 | Exact match вҶ’ 50, same family вҶ’ 35, adjacent вҶ’ 20, intern вҶ’ 10 |
| `LOCATION_MATCH` | 30 | Exact вҶ’ 30, same country вҶ’ 15, remote вҶ’ 5 |
| `INFO_QUALITY` | 20 | +5 each: real salary, explicit experience, вүҘ3 skills, confirmed URL |

```bash
# 1. Clone the repository

cd Job_Search-_Agent

# 2. Create and activate a virtual environment
python -m venv .venv

pip install -r requirements.txt
```

### Configuration

Copy the `.env` template and fill in your keys:

```bash
# .env
GROQ_API_KEY=gsk_xxxxxxxxxxxxxxxxxxxx
TAVILY_API_KEY=tvly-xxxxxxxxxxxxxxxxxxxx

# Model selection (default: LLaMA 3.3 70B on Groq)
GROQ_MODEL=llama-3.3-70b-versatile

# Tuning knobs
MAX_AGENT_ITERATIONS=3    # Recommended ceiling вҖ” don't exceed without reason
```

```bash
uvicorn backend.main:app --reload --host 0.0.0.0 --port 8000
```

Then open **[http://localhost:8000](http://localhost:8000)** in your browser. The static frontend is served automatically by FastAPI.

## вңҰ Design Decisions & Engineering Notes

**Why LangGraph over a simple chain?**
The job-search workflow is inherently stateful: the agent must fire two parallel tool calls, detect whether the internship query ran, inject a coercion message if it didn't, and handle graceful degradation when the iteration cap fires mid-tool-call. A state graph with explicit routing logic is far more auditable than a prompt-jailbreak approach.

**Why a deterministic skill harvester in the parser?**
LLMs hallucinate skills. Pre-extracting a canonical list from the CV text and injecting it verbatim into the prompt (rather than asking the LLM to "read the CV and identify skills") eliminates the main source of phantom-skill-based bad queries.

**Why 8KB for the Wuzzuf/LinkedIn live probe?**
The closure badge and `posted-X-ago` string both render in the first few hundred bytes of the page's `<main>` content вҖ” well before any `<aside>`, `<footer>`, or sidebar widget that would introduce false positives. 8KB reliably captures both while staying safely short of company-bio bleed on Wuzzuf and repost-badge noise on LinkedIn.

**Why is `MAX_AGENT_ITERATIONS=3` the recommended ceiling?**
Groq's API enforces per-minute token limits. At 3 iterations (1 LLM call + 2 tool executions + 1 finalisation call), the agent stays comfortably within free-tier rate limits while covering both the full-time and internship query paths.
